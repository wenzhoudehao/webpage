# TinyShip 会员体系建设实施计划

## 一、需求概述

将原 SaaS 会员系统改造为企业内部权限管理系统，实现以下角色分组：

| 角色 | 权限范围 |
|------|----------|
| **业务会员** | 全部功能，包括金额、收款、客户信息 |
| **生产会员** | 订单管理，但不能接触金额、收款、客户具体信息 |
| **供应商** | 只能接触采购功能模块 |
| **外部客户** | 只能查看自己的订单和推送的款式图片 |

**重要原则**：不能单纯靠隐藏网页来限制功能，必须在前后端都进行权限控制。

---

## ⚠️ 核心原则：真权限 vs 假权限

### 什么是"假权限"（不安全）

```
用户 → 前端隐藏菜单 → 猜到URL → 直接访问页面 → API 正常返回数据 ❌
```

**问题**：前端隐藏只是"掩耳盗铃"，攻击者可以直接调用 API 获取数据。

### 什么是"真权限"（安全）

```
用户 → 前端隐藏菜单 → 猜到URL → 直接访问页面 → API 返回 403 拒绝 ✅
```

**保护层级**：

| 层级 | 位置 | 作用 | 必须实现 |
|------|------|------|----------|
| **L1: API 层** | 后端 route.ts | 验证权限，返回 403/401 | ✅ 必须 |
| **L2: 中间件层** | middleware.ts | 路由级别拦截 | ✅ 必须 |
| **L3: 页面层** | page.tsx | 无权限时重定向 | ⚠️ 推荐 |
| **L4: 组件层** | 菜单/按钮 | 隐藏无权限操作 | ⚠️ 推荐 |

**实施要求**：
- **L1 + L2 必须实现**：这是真正的安全防线
- **L3 + L4 是用户体验优化**：避免用户看到无权限的选项

### Phase 3: 权限规则定义

#### 文件: `libs/permissions/abilities.ts`

**修改内容**:
```typescript
import { AbilityBuilder, createMongoAbility, MongoAbility } from '@casl/ability';
import { Action, AppUser, Role, Subject } from './types';

export type AppAbility = MongoAbility<[Action, Subject | 'all']>;

export function defineRulesForUser(user: AppUser): AppAbility {
  const { can, cannot, build } = new AbilityBuilder<AppAbility>(createMongoAbility);

  switch (user.role) {
    case Role.ADMIN:
      // 管理员拥有所有权限
      can(Action.MANAGE, Subject.ALL);
      break;

    case Role.BUSINESS:
      // 业务会员 - 全部业务权限
      can(Action.MANAGE, Subject.PO);
      can(Action.MANAGE, Subject.PAYMENT);
      can(Action.MANAGE, Subject.VERIFICATION);
      can(Action.MANAGE, Subject.CUSTOMER);
      can(Action.MANAGE, Subject.SUPPLIER);
      can(Action.MANAGE, Subject.PURCHASE);
      can(Action.MANAGE, Subject.PRODUCT);
      can(Action.SYNC, Subject.SYNC);
      can([Action.READ, Action.UPDATE], Subject.USER);
      break;

    case Role.PRODUCTION:
      // 生产会员 - 不能看金额、收款、客户
      can([Action.READ, Action.UPDATE], Subject.PO);
      can([Action.READ, Action.UPDATE], Subject.PRODUCT);
      // 明确禁止
      cannot(Action.READ, Subject.PAYMENT);
      cannot(Action.READ, Subject.VERIFICATION);
      cannot(Action.READ, Subject.CUSTOMER);
      cannot([Action.CREATE, Action.UPDATE, Action.DELETE], Subject.PO, { field: 'totalAmount' }); // 不能修改金额字段
      cannot([Action.CREATE, Action.UPDATE, Action.DELETE], Subject.PO, { field: 'outstandingBalance' });
      break;

    case Role.SUPPLIER:
      // 供应商 - 只能看采购
      can([Action.READ, Action.CREATE, Action.UPDATE], Subject.PURCHASE);
      can(Action.READ, Subject.PRODUCT);
      break;

    case Role.CUSTOMER:
      // 外部客户 - 只能看自己的订单（资源级控制）
      can(Action.READ, Subject.PO);
      can(Action.READ, Subject.PRODUCT);
      // 限制只能看自己的数据（需要在运行时检查）
      break;

    case Role.USER:
    default:
      // 普通用户 - 最小权限
      can(Action.READ, Subject.USER);
      break;
  }

  return build();
}

export function createAbilityForUser(user: AppUser): AppAbility {
  return defineRulesForUser(user);
}
```

### Phase 4: 工具函数更新

```typescript
/**
 * 检查用户是否可以访问特定金额字段
 */
export function canAccessFinancialData(user: AppUser | null): boolean {
  if (!user) return false;
  return [Role.ADMIN, Role.BUSINESS].includes(user.role);
}

/**
 * 检查用户是否可以访问客户信息
 */
export function canAccessCustomerData(user: AppUser | null): boolean {
  if (!user) return false;
  return [Role.ADMIN, Role.BUSINESS].includes(user.role);
}

/**
 * 获取资源级过滤条件（用于外部客户）
 */
export function getResourceFilter(user: AppUser, subject: Subject): Record<string, any> | null {
  if (user.role === Role.CUSTOMER) {
    switch (subject) {
      case Subject.PO:
        return { customerId: user.id }; // 只能看自己的订单
      case Subject.PRODUCT:
        return { isPublic: true }; // 只能看公开的产品
      default:
        return { ownerId: user.id };
    }
  }
  return null;
}

/**
 * 映射数据库角色到权限系统角色
 */
export function mapDatabaseRoleToAppRole(dbRole: string): Role {
  switch (dbRole) {
    case 'admin': return Role.ADMIN;
    case 'business': return Role.BUSINESS;
    case 'production': return Role.PRODUCTION;
    case 'supplier': return Role.SUPPLIER;
    case 'customer': return Role.CUSTOMER;
    default: return Role.USER;
  }
}
```

#### 5.3 资源级权限控制（外部客户只能看自己的数据）

**这是最关键的安全措施**：即使 API 允许访问，也要在数据库查询时过滤。

**文件**: `apps/next-app/app/api/po/route.ts`

```typescript
import { getResourceFilter, mapDatabaseRoleToAppRole } from '@libs/permissions/utils';
import { Role, Subject } from '@libs/permissions/types';

export async function GET(request: NextRequest) {
  const session = await auth.api.getSession({ headers: request.headers });

  if (!session?.user) {
    return NextResponse.json({ error: '未登录' }, { status: 401 });
  }

  // 创建 AppUser
  const appUser = {
    ...session.user,
    role: mapDatabaseRoleToAppRole(session.user.role)
  };

  // 检查基本权限
  if (!can(appUser, Action.READ, Subject.PO)) {
    return NextResponse.json({ error: '无权限' }, { status: 403 });
  }

  // 获取资源级过滤条件（外部客户只能看自己的订单）
  const resourceFilter = getResourceFilter(appUser, Subject.PO);

  // 构建数据库查询
  let query = db.select().from(cachedPO);

  // 应用资源级过滤
  if (resourceFilter) {
    if (resourceFilter.customerId) {
      query = query.where(eq(cachedPO.customerId, appUser.id));
    }
    if (resourceFilter.isPublic !== undefined) {
      query = query.where(eq(cachedPO.isPublic, resourceFilter.isPublic));
    }
  }

  const orders = await query;
  return NextResponse.json({ orders });
}
```

**安全效果**：
- 外部客户调用 `/api/po` → 返回 200，但只有自己的订单
- 即使猜到其他订单 ID，直接调用 `/api/po/rec123` → 返回 404 或空数据

#### 5.4 创建权限中间件

**新建文件**: `apps/next-app/middlewares/permissionMiddleware.ts`

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { auth } from '@libs/auth';
import { can } from '@libs/permissions/utils';
import { Action, Subject } from '@libs/permissions/types';

export async function withPermission(
  request: NextRequest,
  action: Action,
  subject: Subject
) {
  const session = await auth.api.getSession({ headers: request.headers });

  if (!session?.user) {
    return { error: NextResponse.json({ error: '未登录' }, { status: 401 }) };
  }

  if (!can(session.user, action, subject)) {
    return { error: NextResponse.json({ error: '无权限' }, { status: 403 }) };
  }

  return { user: session.user };
}
```
